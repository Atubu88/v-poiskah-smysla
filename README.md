# В поисках смысла

Интерактивный путь о счастье, человеческой нужде, Создателе, Послании, Коране и доверии Богу. Один код работает в обычном браузере и как Telegram Mini App.

## Что входит

- единый маршрут из 8 последовательных этапов;
- 48 интерактивных шагов;
- прямой разговор с пользователем через наблюдения, вопросы, возражения и примеры;
- личная рефлексия без оценки «правильно/неправильно»;
- логические цепочки, один или несколько ответов и раскрывающиеся карточки;
- постепенное открытие этапов и повторное прохождение;
- локальное сохранение прогресса и приватных ответов;
- светлая и тёмная темы;
- адаптивный интерфейс для телефона и компьютера;
- Telegram BackButton, системная тема, тактильный отклик и действие «Поделиться»;
- полная работа основного пути без Telegram и серверной учётной записи;
- защищённая необязательная статистика Telegram через Cloudflare D1;
- административная сводка прохождения без сохранения личных ответов.

## Маршрут

1. Почему нам всегда мало?
2. Ты точно никому не поклоняешься?
3. Когда исчезает контроль.
4. Что говорит этот мир?
5. Кто действительно достоин поклонения?
6. Почему мы должны поверить Посланнику?
7. Вызов, который остаётся.
8. Что значит довериться Богу?

Классической викторины и прежних 10 расследований в этой версии нет.

## Локальный запуск

Нужен Node.js 22.13 или новее.

    npm install
    npm run dev

Vite напечатает локальный адрес. Обычно это http://localhost:3000 или следующий свободный порт.

Проверки:

    npm run lint
    npm run build
    npm run test
    npm run validate:artifact

## Основные файлы

- app/page.tsx — интерфейс, навигация, интерактивы и прогресс;
- app/globals.css — визуальная система и мобильная адаптация;
- lib/journey.ts — восемь этапов и 48 последовательных шагов;
- app/layout.tsx — метаданные и Telegram Web App SDK.

## Хранение данных

Ответы и полный локальный прогресс сохраняются на устройстве под ключом meaning-journey-v2. Их можно удалить в разделе «О пути».

При запуске внутри Telegram сервер проверяет подписанный `initData`. В D1 сохраняются Telegram ID, время запуска бота, первое и последнее открытие Mini App, счётчик подтверждённых открытий и технические отметки завершённых этапов и шагов. Имя, username и выбранные пользователем ответы или размышления в D1 не сохраняются.

Команды администратора:

- `/users_count` — пользователи бота и подтверждённые открытия Mini App;
- `/progress` — общая сводка прохождения и число завершивших каждый этап;
- `/progress <telegram_id>` — количество этапов и шагов конкретного пользователя без его ответов.

Команды доступны только Telegram ID из `TELEGRAM_ADMIN_IDS`.

## Настройка Telegram и D1

Обязательные production-переменные:

- `TELEGRAM_BOT_TOKEN` — токен бота, хранится как Cloudflare secret;
- `TELEGRAM_WEBHOOK_SECRET` — случайный секрет webhook, хранится как Cloudflare secret;
- `TELEGRAM_ADMIN_IDS` — Telegram ID администраторов через запятую;
- `MINI_APP_URL` — публичный HTTPS-адрес приложения;
- `TELEGRAM_INIT_DATA_MAX_AGE_SECONDS` — срок действия initData, по умолчанию 3600 секунд.

Пример локальных имён находится в `.dev.vars.example`. Настоящие значения нельзя добавлять в Git.

Безопасный порядок обновления без остановки существующего бота:

1. Добавить новые secrets и переменные в Cloudflare.
2. Применить D1-миграции.
3. Зарегистрировать webhook с тем же `TELEGRAM_WEBHOOK_SECRET`.
4. Только после этого развернуть новую версию Worker.

Команда миграции:

    npx wrangler d1 migrations apply v-poiskah-smysla-db --remote

Команда регистрации защищённого webhook:

    npm run telegram:set-webhook

Старая таблица `telegram_users`, если она существует, автоматически переносится в обезличенную `telegram_usage`. Имена и username из старой таблицы очищаются. Прогресс хранится отдельно в `telegram_progress`, `telegram_completed_stages` и `telegram_completed_tasks`.

## Редакторская граница

Новые религиозно содержательные формулировки требуют ручной проверки перед публикацией. Личные ответы пользователя не оцениваются как показатель его веры.
